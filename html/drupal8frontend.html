1How to include javascript file in Drupal 8?

In Drupal 8 Libraries are simply collections of CSS or JS files bundled together under a uniquely identified library name. We can include the library in the theme-name.libraries.yml file in the below method.

    
    library-name:
      js:
        js/script.js: {}

2How to create a custom theme in Drupal 8?

  • Create a folder with the theme name inside the themes folder from the roor directory.
  • Create a file with a name theme_name.info.yml. ss
  • theme_name.info.yml file will be look like the below
              
  name: 'theme_name'
  type: theme
  description: 'Custom drupal 8 theme'
  stylesheets:
    all:
      - css/styles.css
  js:
    js/main.js: {}
  regions:
    header: 'Header'
    messages: 'Messages'
    navigation: 'Navigation'
    sidebar: 'Sidebar'
    sidebar_two: 'Sidebar Two'
    content: 'Main content'
    footer: 'Footer'
  features:
    - favicon
    - logo
  version: 8.0.x
  core: 8.x

3How to declare the css and js in the libraries.yml file?

Create library-name.libraries.yml file in the themes folder and add the below line 
             
  base:
  version: 1.x
  css:
    theme:
      css/style.css: {}
    layout:
      https://www.example.com/css/main.css: { type: external, minified: true }
  js:
    js/main.js: {}
  dependencies:
    - core/drupal
    - core/jquery
    - core/jquery.once
Finally mention the below code in the theme_name.info.yml file. 
  libraries:
    theme_name/base

4How many ways to attach the js file from the libraries.?

You can attach an asset library to a Twig template using the attach_library() function in any *.html.twig, file like so: 

    
  {{ attach_library('theme_name/base') }}
  markup {{ message }}

In the theme hook function add the below code 

    
  function themename_preprocess_page(&$variables) {
    $variables['#attached']['library'][] = 'theme_name/base';
  }

16List some url related function in twig.

             
  file_url($uri) :

    This helper function accepts a URI to a file and creates a relative URL path to the file.

    {{ file_url(node.field_example_image.entity.uri.value) }}

  link($text, $uri, $attributes) :

    This helper function accepts as first parameter the text and as second parameter the uri

    Examples:
      {{ link(item.title, item.uri, { 'class':['foo', 'bar', 'baz']} ) }}

  path($name, $parameters, $options) :

    Generates a [relative] URL path given a route name and parameters.

    {# Link to frontpage view. #}
    <a href="{{ path('view.frontpage.page_1') }}">{{ 'View all content'|t }}</a>

    {# Link to user entity/profile page. #}
    <a href="{{ path('entity.user.canonical', {'user': user.id}) }}">{{ 'View user profile'|t }}</a>

    {# Link to node page. #}
    <a href="{{ path('entity.node.canonical', {'node': node.id}) }}">{{ 'View node page'|t }}</a>

  url($name, $parameters, $options) :

    Generate an absolute URL given a route name and parameters:

    <a href="{{ url('view.frontpage.page_1') }}">{{ 'View all content'|t }}</a>

    Generate an absolute URL to the current url:

    <a href="{{ url('') }}">{{ 'Reload'|t }}</a>

    Generate an absolute URL to the front page:

    <a href="{{ url('') }}">{{ 'Home'|t }}</a>

17How to suggest the custom template file from controller page?

If we have a twig template file called 'view-env-variables.html.twig', then we can suggest this template file from the controller by the below method, 
              
  namespace Drupal\env_variables\Controller;

  use Drupal\Core\Controller\ControllerBase;

  /**
    * Controller routines for env_variables routes.
  */
  class ViewEnvironmentVariablesController extends ControllerBase {

    public function view_env_variables() {
      return [
        '#theme' => 'view_env_variables',
        '#result' => $_ENV,
      ];
    }
  }
